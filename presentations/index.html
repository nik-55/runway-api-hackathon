ReelAgent

AI Reaction Reel Generator — Runway API Hackathon

Paste a YouTube URL. Get a finished reaction reel with a lip-synced character commentator. No editing. No prompting individual tools.

  • YouTube videos are long — most of the value lives in 30 seconds
  • Paste a URL, optionally type a direction, and the agent handles everything
  • An AI picks the sharpest moment, writes a reaction take, and calls Runway to generate the assets
  • A lip-synced avatar character delivers the commentary — no filming, no editing
  • Output is a finished reel, ready to post

get_frames local · vision sub-call
▶
ffmpeg extracts frames from the source video, passes them to a Kimi vision sub-call with a text prompt, returns text only. Raw images never reach the orchestrator — keeps main context lean. Used when transcript alone is ambiguous.
Model: Kimi K2.6 (same HF endpoint, separate completion)
generate_reaction_image Runway
▶
Generates a still meme-style or reaction image. Used when the agent wants a punchy static visual cut — a bold text card, a reaction face, a meme beat. 9:16 vertical.
Model: gen4_image
generate_animated_reaction Runway
▶
Generates a short text-to-video reaction clip — the equivalent of a reaction GIF but created fresh for this moment. Can also be used as an overlay on top of the source (e.g. cartoon "NO", exploding emoji, money flying). Duration: 4, 6, or 8 seconds.
Model: gen4.5
generate_sound_effect Runway
▶
Generates an audio sting — record scratch, vine boom, dramatic reverb, comedic hit. Layered under a cut to punctuate the joke or the reveal.
Model: eleven_text_to_sound_v2
generate_character_video Runway · always called
▶
Generates a lip-synced avatar video of the commentary script. Script includes inline emotion markers — [shocked], [laughing], [sarcastic], [deadpan], etc. — that score how each line is delivered. The agent must call this exactly once.
Model: gwm1_avatars (avatar_videos)
isolate_voice Runway · always called
▶
Slices the source audio with ffmpeg then runs voice isolation on it. Required for any window of original audio kept audible in the reel — removes background noise and music.
Model: eleven_voice_isolation
finalize_reel ends the loop
▶
The agent submits its full assembly plan. Pydantic validates it — tracks must tile from 0 with no gaps, all asset_ids must exist, total duration must be in range. On valid: writes plan.json and exits the loop. On invalid: returns the error so the agent corrects itself.
Backend: pydantic validation → plan.json → ffmpeg assembly
